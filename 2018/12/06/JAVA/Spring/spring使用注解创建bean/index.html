spring使用注解创建bean

发表于2018-12-06|更新于2022-04-06|Spring
|评论数:

配置

告知spring在创建容器时要扫描的包，配置所需要的标签不是在beans的约束中，而是一个名称为context名称空间和约束中

<context:component-scan base-package="com.itheima"></context:component-scan>

然后对应包下面的类就会被扫描,这个时候创建了accountDao1的bean在容器中

注入数据

通过三个注解可以注入bean，Repository Component Controller Services

@Autowired可以自动注入非基本类型数据，它会在ioc中查找相同类型的bean，如果有多个相同类型的bean就会比对id

如果要注入基础类型数据则需要使用@Value注解

完全消除bean.xml

想要完全消除xml，需要使用@Configuration,@ComponentScan("com.itheima")对类进行注释，第一个注解表示这是一个配置类，第二个类表示需要进行扫描的包

@Import注解

  • 作用：用于导入其他的配置类
  • 属性：
    • value：用于指定其他配置类的字节码。
    • 当我们使用Import的注解之后，有Import注解的类就父配置类，而导入的都是子配置类

@PropertySource可以指定配置文件源，然后就注解中就可以使用el表达式

@Scope可以指定单例还是多例

通过AnnotationConfigApplicationContext获取容器，其中传入的类可以不使用@Configuration注解（这里就是SpringConfiguration.class）

@Qualifier当有ioc多个相同类型对象，无法自动注入，可以通过这个注解指定id
