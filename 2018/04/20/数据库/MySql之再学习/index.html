MySql之再学习

发表于2018-04-20|更新于2022-04-06|Mysql
|评论数:

连接数据库

window下启动和停止mysql

​ net start nysql 和 net stop mysql

mysql -h 服务器地址 -P 端口 -u 用户 -p 密码

MYSQL字符编码设置

show variables : 显示系统编码设置

character_set_client : 发送数据的编码

character_set_connection : 转换数据用的编码

character_set_database : 当前数据库的编码

character_set_filesystem : 把os上文件名转化成此字符集，即把 character_set_client转换character_set_filesystem，默认binary是不做任何转换的

character_set_results : 返回数据的编码

character_set_server : 服务器的默认编码

character_set_system : 标识符，数据库名，表名，字段名的编码

character_sets_dir : 字符集安装目录

数据库基本操作

设置临时编码

​ charset gbk;

​ set names gbk;

创建数据库：

​ create database ‘’数据库名’’;

查看当前存在的数据库：

​ Show databases;

查看数据的创建语句：

​ Show create database 数据库名;

数据库删除：

​ Drop database 数据库名;

修改数据库信息

​ Alter database 数据库名 [修改指令]

​ alter database 数据库名 set character gbk;

选择数据库

​ use 数据库名

表的操作

创建表：

​ create table 表名 列选项 表选项;

​ 可以使用前导零填充 zerofill

​ alter table tb_name add c int(2) zerofill

查看当前数据库所有表:

​ show tables;

​ show tables like exam_%(百分号表示任意字符)

显示某个表的创建信息

​ show create table 表名;

显示某个表的结构

​ describe 表名;(简写为 desc 表名;)

​ show columns from 表名;

删除表： 

 drop table [if exists] 表名;

修改表名

​ Rename table old_tbl_name tonew_tbl_name;

修改列定义

​ alter table tbl_name [add|drop|change|modify]

​ 例子:alter table tbl_name add height int

​ alter table tbl_name drop height

​ alter table tbl_name modify height varchar(40)

​ alter table tbl_name change height weight int

数据的操作

插入数据

​ insert into 表名 (字段列表) values (值列表)

获得数据

​ Select 字段列表 from 表名 where 查询条件

删除数据

 Delete from 表名 where 条件

修改数据

​ Update 表名 set 字段=新值, … where 条件

时间类型

日期类型

​ DATATIME ‘2018-04-16 11：50：21’ 8个字节

​ DATE YYYY-MM-DD 3个字节

​ TIME HH:MM:SS 3个字节

​ YEAR YYYY 1个字节

时间戳(存储时是整型，表示时是日期类型)

​ TIMESTAMP ‘2018-04-16 11：50：21’ 4个字节

字符类型

varchar类型

varchar需要一个字节或者两个字节保存字符串总长度

记录存在总长度 65535限制。

varchar特点，当 类型数据超过255个字符时，采用2个字节表示长度。

65535-2=65533

整条记录，需要一个额外的字节，用于保存当前字段的null值。

除非所有的字段都比不是nul，这个字节才可以省略。一个记录，不论有多少个字段

存在null，都是使用统一的一个字节来表示。而不是每个字段一个字节。

Text 2^16 文本，有很多 兄弟类型 Tinytext longtext 表示的字符串长度不一样。

枚举enum

create table test (gender enum(‘female’,’male’));

内部以整型存储，最大值65535

set集合类型

create table test(hobby set(‘basket’,’football’,’baseball’));

basket用二进制1表示，football用10表示，baseball用100表示

列约束

NOT NULL 不为空

COMMENT 注释

DEFALUT 默认值

PRIMARY KEY 主键

UNSIGNED 无符号数

AUTO_INCREMENT 自动增加 插入以最大值的为标准

FOREIGN KEY 外键 Foreign Key (外键字段） references 主表名 (关联字段) 主表记录删除时的动作 主表记录更新时的动作]

级联操作

主表更新

​ on update

主表删除

​ on delete

级联允许的动作（主表发生改变，对从表的操作）

 - cascade  关联操作，主表更新或者删除，从表也会更新删除

 - Set null：设置为null，表示从表不指向任何主表记录。

 - Restrict：拒绝主表的相关操作。(限制，默认值，抛异常)
  • no action: 什么都不操作

alter table 表名 add foreign key (class_id) references 主表 (class_id)

on delete set null;

on delete cascade;

on delete restrict

外键的操作

外键的删除

Alter table tbl_name drop foreign key 外键标识;

删除外键需要通过指定外键名称达到目的：

可以通过在创建外键时，制定名称，

或者使用mysql默认生成的名称

show create table 表名 可以显示外键标识

外键的添加

Alter table tbl_name add [constraint 外键标识] Foreign Key (外键字段） references 主表名 (关联字段) 主表记录删除时的动作 主表记录更新时的动作]

存储引擎

默认的服务器表类型， 通过my.ini可以配置：

default-storage-engine=INNODB

或者在命令行中更改

alter table itcast_class engine myisam;

查询操作

范围比较 between and select * from test where id between 3 and 5 这里是 3，4，5

集合之内 in select * from test where id in(3,5) 这里只有3和5

逻辑运算

非：not !，非null 为null。

与：and &&，有0就是0，都是非零为1，存在null与非零则为null。

或：or || , null||null=null null||1=1 null||0=null

异或： xor ,有null，就是null。

排序 ORDER BY order by 字段 asc|desc(升序|降序)

分组查询 GROUP BY 所有教师的代课天数：

select t_name, sum(days) from teacher_class where 1 group by t_name; 注意配合having使用 必须在group by子句之前指定where子句 WHERE过滤行，HAVING过滤组

限制数量 LIMIT 一般出现在语句最后limit offset(偏移量) , row_count(总记录数) 例子: select * from test limit 10;

去除重复记录 DISTINCT select distinct * from test; 需要注意的是 去除的是所有字段都相同的行记录

UNION 联合查询 使用union可以将两个select查询连接到一起，union默认会消除相同的 选项，此时可以使用 union all 可以获取相同的记录

子查询 子查询就是将查询结果作为表来使用 select t_name,gender from teacher_class where days = (select max(days) from teacher_class) 选择教书天数最多的老师

需要注意的是：子查询可以配合 in any(等于任何一个) all(等于所有的) 来使用

连接查询

  • 内连接 inner join 左表每一个记录和右表每一个进行连接，满足条件保存，不满足则舍去 tb1_left inner join tb1_right on 连接条件
  • 外连接 outer join 进行连接的两个表对应的相匹配的字段可以没有 其中又有 左外连接（不满足条件时保留左边, 就是如果有左边的记录没有合右边的匹配的记录也- 会保留下来） left outer join 右外连接（不满足条件时保留右边）right outer join
  • 全外连接 mysql中没有全外连接，可以使用union模拟，将左外连接和右外连接 使用union进行连接就得到全外连接
  • 自然连接 natural join 自动选取相同的字段作为条件两个进行连接 ，其中自然连接也有 natural left join,natural right join,需要注意的是，相同的字- 段会字段进行合并
  • 笛卡尔积/交叉连接 cross join 没有条件的内连接,默认不写where，即没有条件的内连接就是cross join
  • using(字段) 查找两个表中相同的字段,并且合并两个字段

将记录存入磁盘outfile

select * into outfile ‘d:/‘ from test;

注意的是：不能创建文件夹

记录默认用回车隔开，字段默认用制表符隔开

select * into outfile ‘d:/‘ fields terminated by ‘,‘ lines terminated by ‘\n’ start by ‘start：’ from teacher_class where t_name = ‘小明’

基本操作 insert into 表名 values (数据)

insert into 表名 set 字段名 = ‘属性’

主键冲突条件 duplicate on duplicate key

替换 replace 和insert的唯一区别就是 主键或者索引发生冲突时，自动替换

删除 delete from 表名 where id = 1; 也允许连接 删除

更新 update 表名 set 字段名 = ‘数据’ ; 同样允许连接 更新

mysql -u root -p 数据库名 < d:/数据库名.sql

视图

视图就是在实际的表的基础上抽象出一层，方便用户操作（其实就是封装复杂查询语句）

创建视图 create view 视图名 as select语句

删除视图 drop view (if exists) 视图名;

修改视图 alter view 视图名 as select语句

事务

事务具有原子性atomicity 要么完全提交 要么完全回退 非事务操作都是 set autocommit = 1 并且mysql中只有innodb支持

事务还有 原子性 一致性consistency 隔离性isolation 持久性durability 的特点 简称acid

提交和回退 提交使用commit 回退使用 rollback

开始 事务 start transaction 或者使用begin

触发器

概念 在当前表上，设置一个对每行数据的一个监视器，监听相关的时间，每当事件发生时，会执行一段由sql完成的一段功能代码

创建触发器 create trigger 自定义事件名 时机 事件 执行代码，一个事件只能只能有一个执行代码

create trigger jiaobanfei after update on czbk_student for each row update class set cz_money = cz_money - 20

删除触发器 drop trigger 自定义事件名

查看触发器 show create trigger 自定义事件名

事件和时机 支持 insert delete update 三种事件 ， 时机有 after 和 before

old 和 new 使用old和new可以获取 update更新前和更新后的数据，insert事件不能使用old，delete事件不能使用new

多条更新语句 delimiter 可以改变结尾符

delimiter $$

create trigger ruxue after insert on czbk_student

for each row

begin

update class set count = count + 1

update class set monry = money + 20

end

$$

delimiter ;

索引

  • 单列索引：即一个索引只包含单个列，一个表可以有多个单列索引

  • 组合索引：一个索引包含多个列

    索引要建立在where条件中使用的字段，并且优先在前面使用

查看索引

show index from 表名

创建索引

create index indexName on 表名(字段名)

删除索引

drop index [indexName] on 表名
