node 特性，事件驱动，非阻塞 IO，单线程

个人感觉其实就是，异步和单线程嘛，单线程有利有弊，节约内存，但是一个死全家死，不太健壮，但是这些都有相应的解决方案

常用 API

module 模块实现原理

首先有几个参数
  • id：该模块的唯一标识，没有被引用时是一个点,被别的模块引用就是绝对路径
  • filename：当前模块绝对路径

  • parent：最先引用该模块的模块。

  • children：被该模块引用的模块对象

  • loaded：模块是否已经加载完成，或正在加载中
  • paths：模块的搜索路径
    [ 'D:\\VUE项目\\test\\node_modules',
    ‘D:\VUE 项目\node_modules’,
    ‘D:\node_modules’ ]

events 模块

  1. 可以继承 events 模块，对自己的代码进行事件处理

Process 模块

  • Process.argv：命令行的参数，是一个数组
  • Process.env：可以获取系统的环境变量
  • Process.exit：退出程序
  • process.stdout.write 输出
  • process.stdin.on(‘data’,function(){})
  • process.stdin.pipe(process.stdout)
  • process.kill(id) 杀死进程
  • process.pid 进程 id
  • process.cwd() 当前目录
  • process.chdir(‘..’) 改变当前所在目录
  • process.arch: 系统的架构，64还是32之类的
  • process.platform: 系统的平台
  • process.HOME/USERPROFILE: 用户目录

Stream 模块

  • 标准输入输出

stream根据功能分为很多种，可读流，可写流，读写流，转换流等等

  • 输出流
