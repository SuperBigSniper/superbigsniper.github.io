什么是设计模式

设计模式（Design pattern）代表了最佳的实践，通常被有经验的面向对象的软件开发人员所采用。设计模式是软件开发人员在软件开发过程中面临的一般问题的解决方案。这些解决方案是众多软件开发人员经过相当长的一段时间的试验和错误总结出来的。

设计原则

  1. 开闭原则：对扩展开放，对修改关闭
  2. 里氏转换原则：子类继承父类，单独调完全可以
  3. 依赖倒转原则：引用对象，但这个对象有底层类型，则直接引用底层对象
  4. 接口隔离原则：每个接口应该只是一种角色
  5. 合成/聚合复用原则：新的对象应使用一些已有的对象，使之成为新对象的一部分
  6. 迪米特原则：一个对象应该对其它对象有尽可能少的了解

设计模式之单例模式

单例就是抱着一个类只有一个实例，实现的方法一般是先判断实例是否存在，存在则直接返回，不存在则创建再返回，保证一个类只有一个实例对象

作用

  1. 模块间的通信
  2. 系统中的某个类的对象只能存在一个
  3. 保护自己的属性和方法

设计模式之建造者模式

使一个复杂对象的构建和表示相分离，让同样的构建过程可以创建不同的表示。就是说隐藏了建造的细节，用户只需要知道建造的类型，实际上就是一个指挥者，一个建造者，前者指挥后者

作用

  1. 分布创建一个复杂对象
  2. 解耦封装过程和具体创建的组件
  3. 无需关系组件如何组装

设计模式之工厂模式

定义一个创建对象的接口，这个接口由子类决定实例化哪一个类，该模式使一个类的实例延迟到子类。而子类可以重写接口方法以便创建的时候指定自己的对象类型(抽象工厂)

作用

  1. 对象的构建十分复杂
  2. 需要依赖具体环境创建不同实例
  3. 处理大量具有相同属性的小对象

设计模式之外观模式

外观模式为子系统的一组接口提供一个一致的界面，此模块定义一个高层接口，这个接口使得子系统更加容易使用，就是说隐藏内部实现，只为外部提供接口

作用

  1. 设计初期，将不同的层分离，比如经典的MVC

设计模式之代理模式

代理模式proxy,为其它对象提供一种代理以控制对这个对象的访问

作用

  1. 远程代理
  2. 虚拟代理（网页渲染图片用占位符代替）
  3. 安全代理
  4. 智能代理（如垃圾回收）

设计模式之观察者模式

又叫发布订阅模式，它定义了一种一对多的关系，让多个观察者对象同时监听某一个主题对象，这个主题对象的状态发生改变就会通知所有的观察者对象，使得它们可以自动更新自己

作用

  1. 支持简单的广播同学，自动通知所有已经订阅过得对象
  2. 目标和观察者存在 一种动态关联，增加了灵活性
  3. 目标和观察者的抽象耦合能够单独扩展和重用

设计模式之策略模式

策略模式定义了算法家族，分别封装，让他们之间可以互相替换，此模式让算法的变化不会影响到使用算法的客户，就是将可变部分抽离出来成算法接口，在接口下 实现不同的算法

设计模式之命令模式

命令模式是一个高内聚的模式，其定义为：Encapsulate a request as an object,there by letting you parameterize clients with different requests,queue or log requests,and support undoable operations.（将一个请求封装成一个对象，从而让你使用不同的请求把客户端参数化，对请 求排队或者记录请求日志，可以提供命令的撤销和恢复功能。）

设计模式之迭代器模式

迭代器提供一种方法顺序访问一个聚合对象中个元素，而又不需要暴露方法中的内部表示

设计模式之责任链模式

使多个对象都有机会处理请求，从而避免请求的发送者和接受者的耦合关系。将该对象连成一条链，沿着链传递，直到有一个请求处理

设计模式之适配器模式

适配器是将一个类的接口转化为客户希望的另一个接口，使得原本由于接口不兼容的变的可以一起使用

设计模式之模板方法

模板方法定义了一个操作中的算法的骨架

设计模式之原型模式

是指用原型实例指向创建对象的种类，并且通过拷贝这些原型创建新的对象
