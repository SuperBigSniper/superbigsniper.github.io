linux 下的 nginx 安装

首先去官网

复制以下内容

根据系统和版本进行修改

baseurl=http://nginx.org/packages/centos/7/$basearch/

将文件保存在 /etc/yum.repos.d/nginx.repo

之后进行 yum install nginx 进行安装即可

ubantu 安装

apt-get install nginx

nginx 常用命令

启动 nginx

nginx -c /etc/nginx/nginx.conf

关闭 nginx

nginx -s stop

优雅停止 nginx -s quit 等待子进程处理完毕再关闭

重读配置文件

nginx -s reload

查看配置是否正确

nginx -t / service nginx configtest

匹配符号
  1. ~ 区分大小写匹配
  2. ~* 不区分大小写匹配
  3. !~和!~*分别为区分大小写不匹配及不区分大小写不匹配
  4. ^~ 以某个字符开头 不是正则匹配，是字符串匹配, 和不带符号一样，但是优先级更高
  5. = 精确匹配

优先级上面
= > ^~ > ~ > ~* > 不带符号 。

nginx 模块

  1. stub_status; : 获取 nginx 的一些信息，必须配置在 server 与 location 下

  2. random_index:on : 随机选取主页

  3. sub_filter '' '' : 替换内容,注意只会替换第一匹配的内容（需要配置sub_filter_once）,还有就是默认是不会缓存的，要缓存需要开启sub_filter_last_modified

  4. limit_req与limit_conn_zone: 请求限制,连接限制

  5. auth_basic: auth 认证

  6. expires 24h：缓存设置，过期时间 24 小时

  7. $http_x_forwarded_for 是一个变量表示代理经过的地址

  8. proxy_buffering on 打开缓存区

  9. proxy_redirect default 重定向，默认不用修改

  10. proxy_set_header Host $http_host 设置代理请求头，add_header设置请求头

  11. secure_link 密匙验证

常见内置变量

  • $time_local ： 用来记录访问时间与时区；
  • $request ： 用来记录请求的 url 与 http 协议；
  • $status ： 用来记录请求状态；成功是 200；
  • $remote_addr 与 $http_x_forwarded_for 用以记录客户端的 ip 地址；
  • $body_bytes_s ent ：记录发送给客户端文件主体内容大小；
  • $http_referer ：用来记录从那个页面链接访问过来的；
  • $host ： 请求主机头字段，否则为服务器名称。
  • $http_user_agent ： 客户端 agent 信息
  • $http_cookie ： 客户端 cookie 信息
  • $request_method ： 客户端请求的动作，通常为 GET 或 POST。
  • $content_length ： 请求头中的 Content-length 字段。
  • $content_type ： 请求头中的 Content-Type 字段。
  • $request_uri ： 包含请求参数的原始 URI，不包含主机名，如：”/foo/bar.php?arg=baz”。
  • $uri ： 不包含请求参数的当前 URI，$uri 不包含主机名，如”/foo/bar.html”。
  • $document_uri ： 与$uri 相同。
  • $query_string
  • $args ： 这个变量等于请求行中的参数，同$query_string
  • $document_root ： 当前请求在 root 指令中指定的值。
  • $limit_rate ： 这个变量可以限制连接速率。
  • $request_body_file
  • $request_filename ： 当前请求的文件路径，由 root 或 alias 指令与 URI 请求生成。
  • $remote_addr ： 客户端的 IP 地址。
  • $remote_port ： 客户端的端口。
  • $remote_user ： 已经经过 Auth Basic Module 验证的用户名。
  • $scheme ： HTTP 方法（如 http，https）。
  • $server_protocol ： 请求使用的协议，通常是 HTTP/1.0 或 HTTP/1.1。
  • $server_addr ： 服务器地址，在完成一次系统调用后可以确定这个值。
  • $server_name ： 服务器名称。
  • $server_port ： 请求到达服务器的端口号。

重定向

通过内置变量来重定向到 pc 端与移动端

重写地址使用rewrite

  • last 停止 rewrite 检测, 会重新进 nginx 匹配
  • break 停止 rewrite 检测,只会修改地址，会继续往下走，一般和 proxy_pass 配合就是 last 一样了
  • redirect 返回 302
  • permanent 返回 301

还有代替写法return 301 https://$host$request_uri;

负载均衡

负载均衡可以设置很多参数(记得在 location 中加上include proxy_params)

  • max_fails: 允许请求失败的次数
  • fail_timeout: 经过 max_fails 失败后，服务暂停的时间
  • max_conns: 限制最大的接收连接数

例如server 127.0.0.1:7878 max_fails=1 fail_timeout=10s max_conns=1

  1. 热备(如果你有 2 台服务器，当一台服务器发生事故时，才启用第二台服务器给提供服务。服务器处理请求的顺序：AAAAAA 突然 A 挂啦，BBBBBBBBBBBBBB..)
  1. 轮询（nginx 默认就是轮询其权重都默认为 1，服务器处理请求的顺序：ABABABABAB….）
  1. 加权轮询
  1. ip_hash:nginx 会让相同的客户端 ip 请求相同的服务器。
  1. url_hash 根据 url 的 hash 结果进行分配请求
  1. least_conn 最少连接数，机器连接数量少的分发
  1. 其它配置项
  • down，表示当前的 server 暂时不参与负载均衡。
  • backup，预留的备份机器。当其他所有的非 backup 机器出现故障或者忙的时候，才会请求 backup 机器，因此这台机器的压力最轻。
  • max_fails，允许请求失败的次数，默认为 1。当超过最大次数时，返回 proxy_next_upstream 模块定义的错误。
  • fail_timeout，在经历了 max_fails 次失败后，暂停服务的时间。max_fails 可以和 fail_timeout 一起使用。
  • max_conns 限制最大的接收的连接数
