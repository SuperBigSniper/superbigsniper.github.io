数据结构

发表于2018-03-09|更新于2022-04-06|Python
|评论数:

数据结构是相互之间存在一种或多种特定关系的数据元素的集合

数据结构分类

数据结构分为逻辑结构和存储结构

逻辑结构有：集合结构（仅同属一个集合），线性结构(一对一，栈，队列，数组等)，树形结构(一对多)，图形结构（多对多，网状结构）

存储结构有：顺序结构（顺序映像），链式结构（非顺序映像）,索引结构，散列结构

数据结构的基本概念

  1. 数据：所有能被计算机识别、存储和处理的符号的集合（包括数字、字符、声音、图像等信息 ）。
  2. 数据元素:是数据的基本单位，具有完整确定的实际意义。在计算机程序中通常作为一个整体进行考虑和处理。一个数据元素可由若干个数据项组成。
  3. 数据项:构成数据元素的项目。它是数据不可分割的最小单位。
  4. 数据类型：指一个类型和定义在这个类型上的操作集合。例：C 语言（基本类型：整型、浮点型、字符型等构造类型：数组、结构、联合、指针、枚举等）
  5. 抽象数据元素：抽象定义的、没有实际含义的数据元素。
  6. 抽象数据类型 ADT：用户自己定义的数据类型。

时间复杂度和空间复杂度

频度是指语句重复执行的次数

栈和队列

栈 STACK： 先进后出

栈的出栈规律：如果入栈是从小到大，在出栈序列中任选一个，这个值到最小值（只看这个值后面）一定是从大到小排列

队列 QUEUE：先进先出

队列的假溢：假溢是指队列在达到下标最大位置之后队列空间感觉用尽了，但是在数组的低端还有空闲空间，循环队列就是解决这个问题的。

数组和广义表

压缩存储：为多个值相同的非零元素只分配一个存储空间；对零元素不分配空间

对于对称矩阵来说,只有算出前 i-1 行的总数就可以求出 dii 行第一个的值，加上 j 就可以求得 aij

第 i 行 j 列 的下标为 k = i(i-1)/2+j-1

稀疏矩阵 第一个是横坐标，第二个是纵坐标，第三个是值

( (1,2,12), (1,3,9), (3,1,-3), (3,6,14),

​ (4,3,24), (5,2,18), (6,1,15), (6,4,-7) )

广义表 广义表是线性表的推广。

L=(a1,a2,…,an ),n≥0,ai 可以是单元素，也可以是一个表。类似 pyhton 中的列表

**表头(Head)**：第一个元素

**表尾(Tail)**：除第一个元素外其余元素构成的表

广义表的深度和长度 长度指的是包含多少个元素，深度指的是一个元素中最多嵌套了多少层

二叉树

树的基本术语

  1. 树的结点:包含一个 DE 和指向其子树的所有分支;
  2. 结点的度:一个结点拥有的子树的个数,度为零的结点称为叶结点;
  3. 树的度:树中所有结点的度的最大值 Max(D(I))
    含义:树中最大分支数为树的度;
  4. 结点的层次及树的深度:根为第一层,根的孩子为第二层,若某结点为第 k 层,则其孩子为 k+1 层.
    树中结点的最大层次称为树的深度或高度
  5. 森林:是 m(m=0)棵互不相的树的集合
    森林与树概念相近,相互很容易转换.

结论：一棵树的各结点的度之和，等于该树的分支数目。

二叉树的性质

性质 1. 在二叉树的第 i 层上至多有 2^(i-1)个结点(i≥1)。

​ M 叉树第 i 层有 m^(i - 1)个结点

性质 2. 深度为 k 的二叉树至多有 2^k-1 个结点(k≥1)。（深度一定，二叉树的最大结点数也确定）

​ 深度为 k 的 M 叉树最多有 (M^k-1)/M-1 个结点

性质 3. 二叉树中,终端结点数 n0 与度为 2 的结点数 n2 有如下关系: n0=n2+1

性质 4. 结点数为 n 的完全二叉树，其深度为 log2n + 1

性质 5. 在按层序编号的 n 个结点的完全二叉树中，任意一结点 i(1≤i≤n)有：

⑴ i=1 时，结点 i 是树的根；否则(i1)，结点 i 的双亲为结点 └i/2┘。

⑵ 2i ＞ n 时，结点 i 无左孩子，为叶结点；否则，结点 i 的左孩子为结点 2i。

⑶ 2i+1 ＞ n 时，结点 i 无右孩子；否则，结点 i 的右孩子为结点 2i+1。

性质 6. 含有 n 个结点的二叉链表中，有 n+1 个空指针。(二叉树的节点都有 2 个指针。每个节点有 0 个、1 个或 2 个空指针。对应的有 2 个、1 个、0 个非空指针。非空指针的总数就是二叉树的边的个数。所以空指针就是 2n-(n-1)=n+1 )

树的遍历

  1. 先序遍历： 根左右 根据前序遍历的结果可知第一个访问的必定是 root 结点
  2. 中序遍历：左根右 根据中序遍历的结果，再结合前序遍历的 root 结点去划分 root 结点的左右子树
  3. 后序遍历：左右根 根据后序遍历的结果可知最后访问的必定是 root 结点
  4. 层序遍历：一层一层遍历 可知第一个访问的必定是 root 结点
  5. 树的性质 1：n0 = 1 +n2 + 2n3 + 3n4 + …. + (m-1)Nm
  6. 树的性质 2：第 i 层有 m^(i-1)个结点
  7. 树的性质 3：一共有 m^i-1/m-1 个结点

线索二叉树

  1. 线索二叉树是为了解决 无法直接获得一个结点的前驱和后继
  2. 添加两个指针，LTag 和 RTag
  3. 若结点有左子树，则 lchild 指向左孩子，否则指向前驱，若结点有右子树，则 rchild 指向右孩子，否则指向后继（需要注意的是，这里的前驱后继指的是根据先序中序后序排列之后的）

树的存储

  1. 双亲表示法

    采用一组地址连续的存储单元存储树的结点,通过保存每个结点的双亲结点的位置，表示树中结点之间的结构关系。

  2. 孩子表示法

    通过保存每个结点的孩子结点的位置，表示树中结点之间的结构关系

  3. 孩子兄弟表示法

    用二叉链表作为树的存贮结构。链表的两个指针域分别指向该结点的第一个孩子结点和其右边的下一个兄弟结点

树的遍历

  1. 树的先根遍历等同于对转换所得的二叉树进行先序遍历

  2. 树的后根遍历等同于对转换所得的二叉树进行中序遍历

森林和树的转换

  1. 树转化为二叉树：树的左节点为孩子，右节点为兄弟
  2. 森林转化为二叉树：与树转化基本相同，但是森林左边的树作为顶点

最优二叉树 每次选择最小的两个作为叶节点，约定小的放左边，大的放右边

**二叉排序树(二叉检索树、二叉查找树)BST ** ：折半查找法

二叉排序树或者是一棵空树，或者是具有下列性质的二叉树：

1)若其左子树不空，则左子树上所有结点的值均小于它的根结点的值；

2)若其右子树不空，则右子树上所有结点的值均大于它的根结点的值

3)其左、右子树也分别为二叉排序树

平衡二叉树 AVL ：

定义：

平衡二叉树或者是一棵空树，或者是具有下列性质的二叉树：它的左子树和右子树都是平衡二叉树，且左、右子树的深度之差的绝对值不超过

**二叉排序树的删除运算 **：

  1. 若待删除的结点p 是个叶子结点，则将p 的父结点f 的左孩子指针置空(若p 是f 的左孩子)，或者将f 的右孩子指针置空(若p 是f 的右孩子)

  2. 若待删除的结点p 是仅有一个孩子的非叶子结点，则将p 的左孩子(或右孩子)接为p 的父结点f 的左孩子(若p 是f 的左孩子)，或者将p 的左孩子(或右孩子)接为p 的父结点f 的右孩子(若p 是*f 的右孩子)

  3. 若待删除的结点*p 的两个子树都不为空，则有两种处理方式：

    其一是令 p 的左子树为f 的左子树，而p 的右子树为*s 的右子树；示例

    其二是令 p 的直接前驱或直接后继代替*p，然后再删除其直接前驱(或直接后继)（采用中序排列）

图

基本术语

  1. 邻接点及关联边

    邻接点：边的两个顶点互为邻接点

    关联边：若有边 e= (v, u), 则称顶点 v、u 关联边 e

  2. 有向完全图：具有 n(n-1)个顶点的图，每两个定点具有两根线

  3. 无向完全图：具有 n(n-1)/2 个顶点的图

  4. 网：在图中赋予每条边或弧一定的权值，这种带权的图称为网

  5. 顶点的度、入度和出度 ：

    顶点 V 的出度 = 以 V 为起点有向边数，

    顶点 V 的入度 = 以 V 为终点有向边数，

    顶点 V 的度 = V 的出度+V 的入度

  6. 连通图、强连通图 ：

数组表示法（邻接矩阵） 矩阵的横坐标表示起始点，纵坐标表示终点，值为距离

链表表示法（邻接表） 每个顶点都是一个链表，链表记录了与其相连接的顶点

遍历方法:深度优先遍历 DFS（类似树的先序遍历）和广度优先遍历 BFS（类似树的层次遍历）

连通图 无向图中任意两个顶点都存在路径到达，那么就是连通图，n 个顶点的连通图最少有 n-1 条边

强连通图 有向图中任意两个顶点都存在路径到达，那么就是强连通图，n 个顶点的强连通图最少有 n 条边（环状结构）

连通完全图：指顶点两两之间均有连接，n 个顶点有 n(n-1)/2 条边

强连通完全图：指顶点两两之间均有连接，n 个顶点有 n(n-1)条边

连通分量和强连通分量 就是指图中的部分顶点可以形成连通图，那么这个部分连通图就是一个连通分量或者强连通分量（官方解释：无（有）向图的极大连通子图称为图的连通分量（强连通分量） ）

生成树 ： 一个连通图的生成树是一个极小连通子图，它含有图中全部顶点，但只有足以构成一棵树的 n-1 条边

赋权图 ：指的是路径带权值的图

拓扑序列：如果图中从 V 到 W 有一条有向路径，则 V 一定排在 W 之前。满足此条件的顶点序列为一个拓扑序列

拓扑排序：将有向图中的顶点排成一个序列。

AOE 网：AOE 网(Activity On Edge Network)是边表示活动的网，AOE 网是带权有向无环图

AOV 网：AOV 网（Activity On Vertex NetWork）用顶点表示活动，边表示活动（顶点）发生的先后关系。AOE 网的边不设权值，若存在边则表示活动 a 必须发生在活动 b 之前。

关键活动：若活动 ak 的延迟时间等于 0，则称之为关键活动。

关键路径：从源点到汇点的所有路径中，长度最长的路径叫做关键路径

普里姆(Prim)算法 ：每次选择与顶点相连接的最小的且没有回路的路径

克鲁斯卡尔(Kruskal)算法 ：每次选择权重最小的并且不形成回路的路径，这样既可求得最小生成树

内部排序

稳定排序：所谓稳定的排序算法就是你排序之后相同大小的数值没有发生变化，比如： 2 4 4 1 6 3 排序之后第二 4 的位置依然在一个 4 之后就是他们两个没有发生位置变化;称之为稳定；

直接插入排序

  1. 时间复杂度：O(n2)

  2. 空间复杂度：O(1)

  3. 是一种稳定的排序方法

  4. 适用情况：元素数目少，或者元素的初始序列基本有序

  5. n 在最好情况下(正序)，元素的移动次数为 0，比较次数为 n – 1；

    n 在最坏情况下(逆序)，元素的移动次数为(n+4)(n-1)/2，比较次数为 (n+2)(n-1)/2

希尔排序(Shell’s Sort)也称为缩小增量排序

  1. 时间复杂度：希尔排序的分析是一个复杂问题，增量序列的设置是关键，尚没有正式的依据说明如何设置最佳的增量序列，大量的实验表明希尔排序所需的比较和移动次数可达到 n1.3
  2. 希尔排序的空间复杂度为 O(1)
  3. 是不稳定的排序方法

冒泡排序

  1. 时间复杂度：O(n2)

  2. 空间复杂度：O(1)

  3. 是一种稳定的排序方法

  4. 适用情况：元素数目少，或者元素的初始序列基本有序

  5. n 在最好情况下(正序)，元素的交换次数为 0，比较次数为 n – 1；

    n 在最坏情况下(逆序)，元素的交换次数为 n(n-1)/2，比较次数为 (n+2)(n-1)/2

快速排序

  1. 时间复杂度：O(nlog2n) 最坏情况为 O(n2)

  2. 空间复杂度：O(log2n)

  3. 不稳定的排序方法

  4. 适用情况：快速排序不适合对小规模的序列进行排序

  5. n 快速排序在最好情形下(左、右子区间的长度大致相等)，则结点数 n 与二叉树深度 h 应满足 log2n<h<log2n+1，所以总的比较次数不会超过(n+1)log2n。因此，快速排序的最好时间复杂度应为 O(nlog2n)。理论上已经证明，快速排序的平均时间复杂度也为 O(nlog2n)。

    n 在最坏情况下(逆序或正序)，时间复杂度为 O(n2)

直接选择排序

  1. 时间复杂度：O(n2)
  2. 空间复杂度：O(1)
  3. 不稳定的排序方法
  4. 适用情况：元素数目少的情况，无需完全排序的情况，比如，选出第 i 小的元素
  5. 在 n 个关键字中选出最小者，需要 n-1 次比较，继续在剩余的 n-1 个元素中选出次小者需要 n-2 次比较，依此类推

**堆排序(Heap Sort) **

​ 基本概念：大顶堆指的是 根节点的元素 大于 孩子节点的元素，小顶堆就是根节点小于孩子节点

  1. 时间复杂度：O(nlog2n)
  2. 空间复杂度：O(1)
  3. 不稳定的排序方法
  4. 适用情况：对于记录数较少的序列来说，堆排序的优越性并不明显，但对大量的记录来说堆排序是很有效的

归并排序

  1. 时间复杂度：O(nlog2n)

  2. 空间复杂度：O(n)

  3. 稳定的排序方法

  4. 适用情况：数据量不是特别大的情况，因为空间复杂度太大

  5. n 对于一个具有 n 个元素的序列，元素的移动次数可以使用下面的递归公式计算：

    M(1) = 0

    M(n) = 2M(n/2) + 2n

基数排序

  1. 时间复杂度：O(d(n+rd))，rd 为每个关键字的基数
  2. 空间复杂度：O(rd)
  3. 稳定的排序方法
  4. 适用于元素数目 n 很大且关键字很小的情况
