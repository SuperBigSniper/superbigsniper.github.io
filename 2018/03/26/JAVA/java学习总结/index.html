JAVA学习总结

发表于2018-03-26|更新于2022-04-06|JAVA

java基础注意点

  • 文件名和主类必须一样，main方法必须添加String[] args
  • 构造方法不能有返回值
  • || && 都是短路运算，而 | 与 & 都是非短路运算符号
  • 基本数据类型有
    • 整数 ：byte，short ，int，long（1，2，4，8）
    • 浮点数：float，double（4，8）
    • 布尔 ： Boolean （1位）
    • 字符：char（Unicode编码 2个字节）
  • 创建对象的步骤
    1. 分配对象空间，并将对象变量初始化为0或null
    2. 执行属性值得显示初始化
    3. 执行构造方法
    4. 返回对象的地址给相关的变量
  • 构造函数中调用构造函数小必须使用 this(a, b) 并且必须在第一行
  • 常量，静态变量，静态方法都在堆中的方法区
  • java导入包 重名会 报错，可以使用import java.sql.*，最好使用的时候写全名
  • import static 可以导入 静态变量 比如 import static java.lang.Math.*
  • override中的重写，返回值类型小于等于父类
  • 当父类的属性由private修饰，子类不能访问
    • 从继承的概念来说，private和final不被继承。Java官方文档上是这么说的
    • 从内存的角度来说，父类的一切都被继承(从父类构造方法被调用就知道了
  • 父类的方法被 final 修饰，则子类不能重写，但是可以重载
  • 接口中只能定义常量，默认 public static final 对变量进行修饰，方法使用 public abstract 进行修饰
  • 类只能单继承，接口可以多继承，类可以实现多个接口
  • 内部类（作用域在方法内）
    • 成员内部类
      • 静态内部类 ： 只能访问静态属性方法 new Outer.Inner()
      • 非静态内部类 ： 依托于外部类的实例 new Outer.new Inner()
    • 匿名内部类
      • 就是说有一个接口 A 需要实现，这个时候直接new A(){ 方法体 }，实现方法体

JAVA中的多态

final中的特殊 “多态”

抽象类abstract

  1. 只有抽象类或者接口才能包含抽象方法
  2. 抽象类不能够实例化（因为还有方法没有实现，自然就不能够实例化嘛）
  3. 继承抽象类的子类必须实现抽象类的所有方法，并且被实现的方法可以为抽象方法

接口 interface

接口是为了解决如果父类设置为抽象类，但是一些子类并不需要父类的抽象方法，但是却要重写这个抽象方法

这就使得代码的冗余，所以接口就诞生了，并且还可以通过接口使用 implments 实现多重继承

  1. 接口中所有的方法都没有实现（即没有方法体），即接口中的方法默认都是抽象方法
  2. 接口中的变量会被隐式地指定为public static final变量
  3. 方法会被隐式地指定为public abstract方法且只能是public abstract方法（所以可以显示的加上public和abstract）
  4. 子类继承接口时需要实现所有抽象方法，并且注意修饰符一定为public
  5. 定义接口时如果要加上 public 必须在接口名与文件名相同的条件下

容器之集合

集合是指 存储的都是对象，对象的类型可以不一致，长度可变

List

List类似数组，但是比数组强大，允许元素重复，可以通过索引访问，指定位置插入

ArrayList：实现了可变长的数组，原理是创建新数组和复制数组，查询快，增删慢

LinkedList：采用链表接口保存对象，利于插入和删除对象

Vector: 线程安全的ArrayList，就是相关方法都增加了synchronized

set

set结构有一个很重要的特点，就是元素唯一，不会有重复，且无序

HashSet：不保证元素顺序，底层原理其实还是hashMap，你的值作为key，一个固定对象作为value

TreeSet：可以对集合中的元素进行排序，使用的是二叉树结构，底层一样，只是变为了TreeMap

LinkHashSet：与HashSet不同的是，它维护了一个双向链表，这个链接列表定义了迭代顺序，可以提高查询效率，并且有序

map

map数据结构特点如下

  1. 内部存储都是键值对
  2. 必须保证键的唯一性
  3. map集合没有迭代器，所以也不能使用foreach循环，必须转换为单列集合，list或者set

map特有 put 方法，对键值对进行映射，map.put(“key”,”value”)

需要注意的是put方法有返回值，如果键对应的没有值则返回null，否则返回旧值

map.keySet()：获取map集合中的键的set集合

map.entrySet()：返回包含映射关系的Set集合

map.values()：获取所有的值

一般有三种map

  1. hashtable：数据结构是一个哈希表，数据同步，不允许null作为键值，Properties表示属性集
  2. hashMap：数据结构是一个哈希表，数据不同步，还有LinkedHashMap
    1. 具体算法就是，hash值 = hashcode & (数组长度 - 1)，并且约定长度为2的整数幂
    2. jdk进行了两次散列
    3. 在jdk8中，链表长度大于8，转化为红黑树
  3. treeMap：红黑二叉树结构，不同步，可以对键进行排序,主要用于排序，按照key递增的顺序排序

泛型

泛型是为了让集合方便存储相同的类型，并且随时可以改变存储的类型

生产的字节码文件，也就是class文件是不带泛型的

泛型是用表示，使用 ? 通配符表示任意类型

泛型限定：使用 ? extends Person 表示接受Person的子类，使用 ? super E 表示接受E的父类型

反射

反射是为了解决动态加载，不用每次重写编译，将类的信息写在配置文件中，再用代码去加载

实例对象使用Object的 getClass() 方法获取Class对象

而类本身使用自己的属性 class 表示Class对象

还可以使用 Class.forName() 参数里写类的全名称

Class 对象的 newInstance() 返回对象的实例

Class 对象的 getConstructor(String.class, int.class) 返回的是构造器对象 Constructor，之后再用返回对象的方法 newInstance() 构造对象

Class 对象的 getField(filedName) 获取字段，但是不能获取私有字段

Class 对象的 getDeclareField(filedName) 获取已声明字段，包含私有字段

Class 对象的 getMethod(methodName , String.class , int.class ) 获取方法，第一个参数是方法名字，后面是参数
