安装

npx create-next-app@latest

路由规则

页面都在src/app目录下，layout作为布局页面，通过page.tsx文件名作为页面文件（也可以新建page文件夹，里面怎增加index页面），目录作为url

  • page - 为这个路由提供特定的用户界面。
  • layout - 为这个路由和所有后代路由提供布局UI。
  • loading - 当路由的服务器组件正在加载时，提供一个加载的用户界面(优先级大于页面的suspense)
  • error - 为处理此路由内和下的错误提供UI（除非由子层的另一个错误路由处理）。
  • not-found 404 界面
  • route 会作为接口对外暴露

public下可以存静态文件，http://localhost:3000/1.jpg 直接访问不用带public

app/ui 存放组件，app/lib存放公共函数

page页面会作为children传递给layout

Private folders 私人文件夹

嵌套路由

因为新的目录结构，子目录会使用父目录的layout，作为布局文件，天然实现嵌套

比如有一个hello子目录

访问/hello
/app/layout => /app/hello/layout => /app/hello/page

带参数的路由（动态路由）

/blog/:slug使用下面的目录即可

./app/blog/[slug]/page.tsx

有三种形式

  • [slug]
  • […slug] 相当于获取所有后续的路由参数 例如app/shop/[…slug]/page.tsx访问/shop/a/b/c会得到{ slug: [’a’, ‘b’, ‘c’] }
  • [[…slug]] 和上面基本一样，就是这个动态路由也包括了slug参数为空的情况, 没有参数也可以匹配/shop

路由组

对路由本身没有影响，只是把组织页面放到一起, 并且可以设置不同的layout

例如/app/(shop)/coat/page.tsx路由为/coat, 会加载/app/(shop)/layout.tsx作为布局文件

并行路由(可能需要重启next)

可以使用@符号，增加渲染的页面

children 属性是一个隐式槽，不需要映射到文件夹。这意味着 app/page.js 相当于 app/@children/page.js

当你有app/@analytics/page.tsx和app/@team/page.tsx就可以在app/layout里结构两个参数

如果你访问的路由和并行路由的不匹配比如你访问app/shop，那么就会访问app/@analytics/shop/page和app/@team/shop/page和app/shop/page，如果任何一个没有就会出现返回404

所以这就是并行路由，如果不想给每一个路由都建立多个文件，你可以使用default.tsx
建立app/@analytics/default.tsx和app/@team/default.tsx

这个时候，匹配不到的时候就会渲染default.tsx

Intercepting Routes 拦截路由

通过Link等标签使用的时候，可以拦截路由，不进行跳转，而是进行一个弹窗同时路由会改变

一般是与并行路由一起使用，这样就会优先匹配配置了拦截路由的页面

拦截路由可以使用 (..) 约定来定义，这与相对路径约定 ../ 类似，但针对的是段。

You can use: 您可以使用：

  • (.) to match segments on the same level 匹配同一级别的段
  • (..) to match segments one level above 匹配上一级的段
  • (..)(..) to match segments two levels above 匹配上面两级的段
  • (…) to match segments from the root app directory 匹配根 app 目录中的段
