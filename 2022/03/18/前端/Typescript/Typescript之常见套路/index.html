Typescript之常见套路

发表于2022-03-18|更新于2025-03-08|Typescript
|评论数:

keyof与in运算符

keyof可以把一个对象的所有key解构，in运算符可以把每一个值提取出来

readonly合并

泛型自动推断

因为泛型可以通过参数推断出来，所以可以省略泛型参数

多个函数重载，就可以通过参数自动推断泛型

与交叉类型与条件类型也会有分发条件类型

注意

boolean 其实也是联合类型, 包含true与false

条件类型对 any(unknow会是2，never是never) 做了特殊处理，如果左边是 any，那么直接把 trueType 和 falseType 合并成联合类型返回

递归高阶应用

模式匹配（infer）

例如正则

通过 ts 提供的infer关键字，我们可以使用模式匹配，只能在extends子句使用并且得到的类型只能在为true的子句使用

数组类型的模式匹配

通过...关键字对数组解构

字符串的模式匹配

trim去除空格

函数参数的模式匹配

先是参数的匹配

返回值的匹配

数值运算 （数组中number与length）

通过number将数组解构为联合类型

通过length我们可以获取数组的长度

Add

这个加法类型的思路是，先构建出2个数组，然后将2个数组组合，通过数组的length属性获取数字，这样就可以获得相加之后的值了

Subtract

这个减法类型的思路也差不多，通过infer构建一个减数+结果 == 被减数

Multiply

Divide

StrLen

也是运用了length这个属性，通过string类型匹配前面所有的字符串，infer捕获最后一个，直到字符串为空

重映射

使用as可以将类型进行重映射（4.1之后允许把类型重新映射为子类型）

这里的Key包含了几种三种类型'string | number | symbol', 使用交叉符&可以将类型缩窄

Key & string就会变为string

联合类型转数组

当infer在协变的位置上时，同一类型变量的多个候选类型将会被推断为联合类型，
当infer在逆变的位置上时，同一类型变量的多个候选类型将会被推断为交叉类型。

交叉类型扁平化处理

当交叉在进行展示的时候不是很直观，所以这个时候我们可以定义Simplify来进行扁平化处理

去除never类型

当对象类型里面有never的时候，我们可以通过{...}[keyof T]这样的形式去除掉never
