缓存：
cdn cdn预热 cdn刷新
为什么cdn的域名和主站域名不同，cookie

  • 刷新功能是指提交URL刷新或目录刷新请求后，CDN节点的缓存内容将会被强制过期，当您向CDN节点请求资源时，CDN会直接回源站获取对应的资源返回给您，并将其缓存。刷新功能会降低缓存命中率。
  • 预热功能是指提交URL预热请求后，源站将会主动将对应的资源缓存到CDN节点，当您首次请求时，就能直接从CDN节点缓存中获取到最新的请求资源，无需再回源站获取。预热功能会提高缓存命中率。

错误捕获

  1. 运行时的错误捕获
  • try-catch
  • window.onerror: 两种错误不能捕获，一种是promise抛出的错误，一种是资源加载的错误
  1. 资源加载的错误
  • object.onerror 例如: script可以添加onerror事件，而且不会冒泡（但是可以捕获）
  • performance.getEntries 获取所有的加载资源
  • 还有就是error事件的捕获
  1. 跨域的js错误可以被捕获吗
    可以捕获到，但是不能拿到详细信息，必须做如下设置
  • script增加crossorigin属性
  • 设置js资源响应头Access-Control-Allow-Origin
  1. 错误监控上报
  • ajax
  • image对象上报，创建一个image对象，将要上报的地址赋值给src

判断不同类型的字节object-sizeof

  • null 为0
  • string 为2
  • boolean 为4 (boolean被当做int类型处理)

    使用int的原因是，对于当下32位的处理器（CPU）来说，一次处理数据是32位（这里不是指的是32/64位系统，而是指CPU硬件层面），具有高效存取的特点

  • number 为8
  • symbol
  • object
