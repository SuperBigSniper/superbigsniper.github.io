ffmpeg

发表于2020-03-09|更新于2022-04-06|工具使用
|评论数:

ffmpeg, ffplay, ffmproc

开源的图形界面工具avidemux

  • ffmpeg: 用于操作，转化流媒体内容
  • ffplay: 用于播放流媒体内容 ffplay input.mp4
  • ffprobe: 用于查看流媒体内容详情 ffprobe input.mp4

常见的编码

  • MP4封装：H264视频编码+ACC音频编码
  • WebM封装：VP8视频编码+Vorbis音频编码
  • OGG封装：Theora视频编码+Vorbis音频编码

ffmpeg

-y overide output files，如果存在同名文件，则覆盖
-n never overide output files，如果存在同名文件，则不覆盖
–threads 2 限制线程数

转码音频格式

ffmpeg -i music_flac.flac -acodec libmp3lame -ar 44100 -ab 320k -ac 2 music_flac_mp3.mp3
说明：

  • acodec:audio Coder Decoder 音频编码解码器 （libmp3lame:mp3解码器）
  • ar:audio rate：音频采样率。若不输入，默认用原音频的采样率
  • ab:audio bit rate 音频比特率 若不输入，默认128K
  • ac: aduio channels 音频声道，默认使用原视频的声道

视频压缩

ffmpeg -i video.mp4 -s 1920x1080 -pix_fmt yuv420p -vcodec libx264 -preset medium -profile:v high -level:v 4.1 -crf 23 -acodec aac -ar 44100 -ac 2 -b:a 128k video_avi.avi
说明:

  • s 1920x1080：缩放视频新尺寸(size)
  • pix_fmt yuv420p：pixel format,用来设置视频颜色空间。参数查询：ffmpeg -pix_fmts
  • vcodec libx264：video Coder Decoder，视频编码解码器
  • preset medium: 编码器预设。参数：ultrafast,superfast,veryfast,faster,fast,medium,slow,slower,veryslow,placebo
  • profile:v high :编码器配置，与压缩比有关。实时通讯-baseline,流媒体-main,超清视频-high
  • level:v 4.1 ：对编码器设置的具体规范和限制，权衡压缩比和画质。
  • crf 23 ：设置码率控制模式。constant rate factor-恒定速率因子模式。范围051,默认23。数值越小，画质越高。一般在828做出选择。
  • r 30 :设置视频帧率
  • acodec aac :audio Coder Decoder-音频编码解码器
  • b:a 128k :音频比特率.大多数网站限制音频比特率128k,129k

经过试验，用下面这个是比较合适的，感觉-vcodec libx265很慢

ffmpeg -i video.mp4 -vcodec libx264 -crf 28 compress28.mp4

  • 默认 42s 15.3
  • ultrafast 23s 22m
  • superfast 33s 17.1m
  • veryfast 42s 15m
  • faster 45s 15.2m (h265 1m47 16.1m)
  • slow 45s 15.2m
  • veryslow 2m 14.9m

原始文件 104.8m

  • crf 23 (53s 17m)
  • crf 24 (54s 16.6m)
  • crf 25 (54s 16.2m)
  • crf 26 (50s 15.9m)
  • crf 27 (1m2s 15.6m)
  • crf 28 (1m3s 15.3m)

GPU加速

通过ffmpeg -hwaccels获取具体可以使用的，然后使用下面的命令,其中videotoolbox是前面获取的

ffmpeg -hwaccel videotoolbox -i video.mp4 -vcodec libx264 -crf 28 compress28.mp4

有时候需要指定线程数，可以使用-threads参数

ffmpeg -hwaccel videotoolbox -i video.mp4 -threads 3 -vcodec libx264 -crf 28 compress28.mp4

又有时候需要进行批量转化，可以利用shell脚本

视频码率控制

ffmpeg支持的码率控制模式：-qp -crf -b
(1)
-qp :constant quantizer,恒定量化器模式
无损压缩的例子（快速编码）
ffmpeg -i input -vcodec libx264 -preset ultrafast -qp 0 output.mkv
无损压缩的例子（高压缩比）
ffmpeg -i input -vcodec libx264 -preset veryslow -qp 0 output.mkv

(2)
-crf :constant rate factor,恒定速率因子模式 (0-51) 一般用这个（一般用23就是1080p+，28就是1080p）

(3)
-b ：bitrate,固定目标码率模式。一般不建议使用

3种模式默认单遍编码

VBR(Variable Bit Rate/动态比特率) 例子
ffmpeg -i input -vcodec libx264 -preset veryslow output
ABR(Average Bit Rate/平均比特率) 例子
ffmpeg -i input -vcodec libx264 -preset veryslow -b:v 3000k output
CBR(Constant Bit Rate/恒定比特率) 例子
-b:v 4000k -minrate 4000k -maxrate 4000k -bufsize 1835k

mp4转m3u8 ts

每个切片60秒

ffmpeg/bin/ffmpeg.exe -i 1.mp4 -c:v libx264 -c:a aac -strict -2 -f hls -hls_time 60 output.m3u8

-c:v 就是 -vcodec

  • -hls_time n: 设置每片的长度，默认值为2。单位为秒
  • -hls_list_size n:设置播放列表保存的最多条目，设置为0会保存有所片信息，默认值为5
  • -hls_wrap n:设置多少片之后开始覆盖，如果设置为0则不会覆盖，默认值为0.这个选项能够避免在磁盘上存储过多的 片，而且能够限制写入磁盘的最多的片的数量
  • -hls_start_number n:设置播放列表中sequence number的值为number，默认值为0

合并,提取音视频

(1)单独提取视频（不含音频流）

  • an : disable audio 静音，也就是删除音频流

ffmpeg -i video.mp4 -an -vcodec copy video_silent.mp4

(2)单独提取音频（不含视频流）

  • vn : disable video，删除视频流
    ffmpeg -i video.mp4 -vn -acodec copy video_novideo.m4a

针对性的单一的提取，例如提取第2条，用指令： -map 0:3
(3)合并音视频

ffmpeg -i video_novideo.m4a -i video_silent.mp4 -c copy video_merge.mp4

截取，连接音视频

(1)截取

  • ss 开始时间
  • to 结束时间
  • t 持续时间

ffmpeg -i music.mp3 -ss 00:00:30 -to 00:02:00 -acodec copy music_cutout.mp3
截取60秒
ffmpeg -i music.mp3 -ss 00:00:30 -t 60 -acodec copy music_cutout60s.mp3

-sseof : 从媒体末尾开始截取

ffmpeg -i in.mp4 -ss 00:01:00 -to 00:01:10 -c copy out.mp4
ffmpeg -ss 00:01:00 -i in.mp4 -to 00:01:10 -c copy out.mp4 关键帧技术，会加速
ffmpeg -ss 00:01:00 -i in.mp4 -to 00:01:10 -c copy -copyts out.mp4 加强版，上面那个容易出错

把-ss放到-i之前，启用了关键帧技术，加速操作。但截取的时间段不一定准确。可用最后一条指令，保留时间戳，保证时间准确。

(2)连接音视频
ffmpeg -i "concat:01.mp4|02.mp4|03.mp4" -c copy out.mp4

不同格式的音视频可以连接在一起，但不推荐不同格式连接在一起。
建议使用Avidemux软件连接

截图,水印,动图

(1)截图.
截取第7秒第1帧的画面

  • vframes 设置第一帧
    ffmpeg -i video.mp4 -ss 7 -vframes 1 video_image.jpg

(2)水印

  • filter_complex 一个滤镜
    ffmpeg -i video.mp4 -i qt.png -filter_complex "overlay=20:20" video_watermark.mp4

(3)截取动图

  • r 设置帧率
    ffmpeg -i video.mp4 -ss 7.5 -to 8.5 -s 640x320 -r 15 video_gif.gif

录屏,直播

(1)录屏
windows: ffmpeg -f gdigrab -i desktop rec.mp4
ubuntu: sudo ffmpeg -f fbdev -framerate 10 -i /dev/fb0 rec.mp4

gdigrab ：ffmpeg中的一个组件。
只捕获视频.若要录屏，录音，获取摄像头，麦克风，换组件，用OBS Studio软件

m3u8转MP4

有两种方式

  1. 通过m3u8文件直接转化为

ffmpeg -i 419985.m3u8 -c copy out1.mp4

这样就会把m3u8文件中的视频转化为mp4

  1. 根据文件内容合并

ffmpeg -f concat -safe 0 -i file.txt -c copy out.mp4

  1. 合并加密的m3u8文件

ffmpeg -allowed_extensions ALL -protocol_whitelist "file,http,crypto,tcp" -i xxx.txt -c copy xxx.mp4

将要合并的文件写入一个人文件

注意文件前面需要有file指令
