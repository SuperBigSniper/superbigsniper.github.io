width

width:auto 时会发生的情况

  1. 最小宽度：当父级宽度为0，宽度为auto元素最小宽度也不会为0，汉字为一个字的宽度，英文为终止到空格，短横线，问号和其它非英文字符终止，可以使用word-break:break-all强制一个英文字符换行
  2. 最大宽度：默认是父级的宽度，当有多个内联盒子时且有换行，就是多个内联盒子加在一起的宽度

input是一种特殊情况，width:auto,display:block无法使得input铺满父级元素，需要使用width:100%才可以，但是又会出现input设置padding后，宽度不好控制，所以有两种解决方案，一种是box-sizing,一种是加一个父元素单独写宽度

height

height的auto比较简单，有多少就增加多少

但是height的100%想要占满屏幕却需要在一定情况下才能触发（我直接100vh）

  1. html,body{ height:100% } 就是html与body元素设置为100%。(高度为content的高度)
  2. 使用绝对定位也可以让元素获得height:100%。（高度为content+padding）

max-width/heigth,min-width/height

  1. max-width会覆盖width，并且优先级超过1improtant
  2. max-xxxx默认值为none，min-xxxx默认值为auto

内联元素

ex单位表示字母x的高度，而字母x表示基线，并且内联元素默认基线对齐，当设置图片为1ex，图片相当于就是一个字母x，也就是会默认居中,当然也可以用vertical-align:middle可以让

  1. 设置height为1ex可以快速让内联元素居中

line-hieght

line-height决定了大部分元素（除了input）撑起来的高度，对于块级元素是改变了其内部元素的属性

  1. line-height不同的数值继承性不同，1.5可以被子元素继承，150%与1.5em只继承更元素

vertical-align

  1. 支持数值比如 10px,就在基线往上偏移10px

  2. 只能作用于内联元素与table-cell元素上

  3. 浮动和绝对定位会导致失效

  4. 块级元素嵌套内联元素要保证内外字体大小设置一致，否则会导致line-height与height不一致
    （1）图片块状化。可以一口气干掉“幽灵空白节点” 、line-height 和 vertical-
    align。

    （2）容器 line-height 足够小。只要半行间距小到字母 x 的下边缘位置或者再往上，自
    然就没有了撑开底部间隙高度空间了。比方说，容器设置 line-height:0。

    （3）容器 font-size 足够小。此方法要想生效，需要容器的 line-height 属性值和当
    前 font-size 相关，如 line-height:1.5 或者 line-height:150%之类；否则只会让下
    面的间隙变得更大，因为基线位置因字符 x 变小而往上升了。

    （4）图片设置其他 vertical-align 属性值。间隙的产生原因之一就
    是基线对齐，所以我们设置 vertical-align 的值为 top、middle、
    bottom 中的任意一个都是可以的

css层叠

层叠顺序：backgroud/border -> 负z-index -> block -> float -> inline -> z-index:auto/0 -> 正index

文本

  1. letter-spacing：控制字符间距
  2. word-spaceing：只控制空格间距
  3. word-break：break-all 所有的都换行，毫不留情，一点儿空隙都不放过
  4. word-wrap:break-word 和上面功能差不多，但是会留下空白
  5. white-space 控制空白如何显示
    • normal：合并空白字符和换行符。
    • pre：空白字符不合并，并且内容只在有换行符的地方换行。
    • nowrap：该值和 normal 一样会合并空白字符，但不允许文本环绕。
    • pre-wrap： 空白字符不合并， 并且内容只在有换行符的地方换行， 同时允许文本环绕。
    • pre-line：合并空白字符，但只在有换行符的地方换行，允许文本环绕。
  6. text-overflow:ellipsis 文字内容超出打点效果
