go的goroutine与并发

发表于2020-11-25|更新于2022-04-06|Go
|评论数:

goroutine 的启动

go 语言通过go关键字开启 goroutine，使用runtime.GOMAXPROCS设置最大核心数

协程退出与程序退出
  • runtime.Goexit是退出当前的协程
  • os.Exit是退出当前程序

channel

channel注意点

chanel 有双通 与 只读/只写 channel

channel 的 close 是广播机制

channel有一些注意点

  • 管道没有使用make分配内存，那么这个 channel 就为nil
  • 从 nil 的管道读取或者写入数据都会阻塞，不会崩溃
  • 从一个 close 的channel读取数据，会返回空值(value 为空值，ok 为 false)
  • 从一个 close 的channel写入数据，会崩溃
  • 关闭一个已经关闭的channel, 会崩溃
  • 读与写不对等，会发生死锁
  • 读写channel可以转化为只读只写channel
  1. chan<- int 是一个只写单向通道（只能对其写入 int 类型值），可以对其执行发送操作但是不能执行接收操作；
  2. <-chan int 是一个只读单向通道（只能从其读取 int 类型值），可以对其执行接收操作但是不能执行发送操作

go 奉行通过通信来进行共享内存，而不是共享内存来通信

channel 是 goroutine的沟通的桥梁，大都是阻塞同步
make 开启 close 关闭 (close 关闭的时候会通知所有的 channel，并且会发送一个空值)
channel 是引用类型
可以设置双向单向 通道
可以设置缓存大小，在未填满前不会发生阻塞

chan表示 channel 类型,chan bool表示 bool 型的 channel，还可以有 int 等等

runtime.NumGoroutine() 可以查看当前运行的 goroutine

channel实现生成器
channel实现Future
channel实现超时器
协程错误recover
并发限制
链式协程

select 机制

  • 可处理一个或多个 channel 的发送与接收
  • 同时有多个可用的 channel 时 按照随机顺序处理
  • 可用空的 select 来阻塞 main 函数
  • 可设置超时

select类似switch，不过是专门用于channel的，每次会去判断所有的case语句后的channel能否成功，成功就执行 case 对应的代码块，如果都没有成功就执行default

context

context在 golang1.9 以后加入，便于用户将 channel 进行取消

  • context.Background() 创建根 context
  • context.withCancel(parentContext) 创建子 context, 创建完成会返回 ctx, cancel

当父 context 被取消，子 conext 也会被取消，通过 <-ctx.Done 去进行判断

锁的分类

互斥锁

互斥锁的意思就是同时只能有一个线程拿到该对象

使用sync.Mutex表示

mutex.lock上锁，mutex.unlock解锁

读写锁

读写锁的意思就是，写的时候只能有一个，读的时候可以有多个，当一个 goroutine 获取读锁之后，其他的 goroutine 如果是获取读锁会继续获得锁，如果是获取写锁就会等待；当一个 goroutine 获取写锁之后，其他的 goroutine 无论是获取读锁还是写锁都会等待。

使用sync.RwMutrx表示

mutex.RLock读锁，mutex.RUnlock解读锁，mutex.Lock写锁，mutex.Unlock解写锁

sync.Once

多个 goroutine 并发执行只执行一次

通过once.Do传入回调函数，在所有协程中只会执行一次

观察结果我们可以发现，函数只运行了一次，且所有的地址都是相同的

sync.Map

Go 语言中内置的 map 不是并发安全的

同时 sync.Map 内置了诸如 Store、Load、LoadOrStore、Delete、Range 等操作方法

原子操作

代码中的加锁操作因为涉及内核态的上下文切换会比较耗时、代价比较高。针对基本数据类型我们还可以使用原子操作来保证并发安全，因为原子操作是 Go 语言提供的方法它在用户态就可以完成，因此性能比加锁操作更好。Go 语言中原子操作由内置的标准库 sync/atomic 提供
