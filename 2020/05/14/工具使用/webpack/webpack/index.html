webpack

发表于2020-05-14|更新于2022-04-06|webpack
|评论数:

安装

  • yarn add webpack
  • yarn add webpack-cli（4.0 以后还需要安装 webpack-cli）
  • webpack 可以实现零配置，使用 npx webpack 运行
  • 零配置下，文件必须在 src 目录下,并且入口文件为 index.js

webpack 配置

  • webpack-config.js 作为 webpack 的默认配置文件
  • 可以使用 webpack –config ‘配置文件’ 指定配置文件

环境变量设置

  • webpack.DefinePlugin 可以定义环境变量
  • 根据环境写不同的配置文件，最后通过 webpack-merge 的 smart 进行合并

resolve

用于解析方面的配置

  • modules

devtool 调试

devtool 中有几个选项

  • devtool:'source-map' 会单独生成 sourcmap 文件
  • devtool:'eval-source-map' 不会产生单独的文件，但是可以显示行和列
  • devtool:'cheap-module-source-map' 不会产生列 是一个单独的文件
  • devtool:'cheap-module-eval-source-map'不会产生列 集成到打包文件中

实时打包

  • 通过 watch 选项进行实时的打包
  • 通过 watchOption 进行选项的配置

多个入口，webpack 支持多个入口

多入口就会产生多个chunk,还有import()导入的也会有chunk

插件

html-webpack-plugin
  • html-webpack-plugin 将打包好的 js 自动引入到 html 中
webpack-dev-server
  • webpack-dev-server 专门用来开发时自动打包，刷新等
  • 在 devServer 中配置
clean-webpack-plugin
  • clean-webpack-plugin 清除上次生成的文件
copy-webpack-plugin
  • copy-webpack-plugin 将文件拷贝到指定目录
webpack.BannerPlugin
  • webpack.BannerPlugin 在打包文件中添加声明
webpack.HotModuleReplacementPlugin
  • webpack.HotModuleReplacementPlugin 用来部分刷新，而不是刷新整个页面

代理（跨域）

  • 使用 webpack 自带的跨域设置（本质上是起了一个服务器）
  • 上面的情况是针对没有服务器，直接使用 webpack 作为服务器

loader

expose-loader 与 webpack.ProvidePlugin

  • 暴露全局变量到 window，比如 jquery

-给每一个模块提供一个$，比如 jquery

  • 当使用 cdn 时不需要打包时externals:{jquery:'$'}

css-loader,style-loader

  • 使得 webpack 能够解析并插入 css，css-loader 解析，style-loader 插入

其它 css 预处理

  • less less-loader
  • node-sass sass-loader
  • stylus stylus-loader

将 css 抽离到 css 文件中，而不是 style 标签中

  • extract-text-webpack-plugin@next （webpack4 需要加上@next，3 不需要，可以抽离多个）
  • mini-css-extract-plugin （抽离为多个，则需要多引入几次）
  • 上面两个插件功能相同，然后就不需要 style 标签了

删除多余的 css 样式

  • purifycss-webpack
  • purify-css
  • glob
  • purifycss-webpack 会调用 purify-css,使用 glob 进行搜索

添加样式前缀

  • postcss-loader
  • autoprefixer

打包图片

  • file-loader
  • url-loader 增强了 file-loader,根据图片大小设置是否转化为 base64

es6/7 转 es5

  • babel-loader
  • babel-core|@babel/core （babel-loader 8.x 版本的，使用@babel/core）
  • babel-preset-env|@babel/preset-env (包含 es5,6,7 和最近的新特性,加@是 8.x 版本使用)
  • 上面做完之后并不能转换像 Set 这样的方法，上面只是针对语法,函数与方法需要下面的插件
  • babel-polyfill | @babel/polyfill （填充器，直接将函数放在全局，使用时只需要 import('@babel/polyfill')）
  • babel-runtime-transform|@babel/plugin-transform-runtime（依赖@babel/runtime） （用到才导入）需要在 babelrc 文件中配置
  • 在 babel7 之后 npx babel-upgrade 升级 babel-stage-0 为分散的插件

babel 插件

  • @babel/plugin-proposal-optional-chaining 可选链式调用
  • @babel/plugin-proposal-nullish-coalescing-operator 空值判断/空值联合

eslint

  • 配置 eslint
  • yarn add eslint eslint-loader

优化 Optimization

  • webpack 优化选项
  • noParse 不解析某些包
  • webpack.IgnorePlugin 忽略某些模块
  • webapck.Dllplugin 添加动态链接库
  • happypack 多线程打包

Tree Shaking 树摇

  • 就是说把你引入，但是没有使用的代码删除
  • webpack-parallel-uglify-plugin （js 的 Tree Shaking）
  • purify-css purifycss-webpack

混淆压缩代码

  • webpack.optimize.UglifyJsPlugin
  • new webpack.optimize.UglifyJsPlugin()

自动复制一些文件

  • CopyWebpackPlugin

编译 typescript

  • ts-loader
  • typescript
  • tsconfi.json 配置 ts

提取公有代码

  • webpack.optimize.CommonsChunkPlugin （webpack4 以前使用这个配置）
  • webpack4 将其添加到了配置中 optimization

代码分割和懒加载

  • webpack 的代码分割和懒加载，并不需要配置，而是一种特殊的代码编辑方式
  • ES 2015 Loader (使用 es2015 的动态 import)
  • 需要引入 @babel/plugin-syntax-dynamic-import
  • output 中的 chunkFilename 就是异步代码块的名字
