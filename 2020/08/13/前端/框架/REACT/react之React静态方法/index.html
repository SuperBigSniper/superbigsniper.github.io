react之React静态方法

发表于2020-08-13|更新于2022-04-06|REACT
|评论数:

cloneElement

以 element 元素为样板克隆并返回新的 React 元素

等同于<element.type {...element.props} {...props}>{children}</element.type>

  • 第一个参数：用于克隆的母体 React 元素。
  • 第二个参数：为克隆后生成的 React 元素添加新的 props 或覆盖从母体中克隆而来的部分或全部 props。
  • 第三个参数：为新生成的 React 元素添加新的 children，取代从母体中克隆而来的 children。

isValidElement

React.isValidElement(object)

验证对象是否为 React 元素，返回值为 true 或 false。

Children

主要是解决当 children 是 单一对象 时, React.Children.xxx 可以正常计算 count, 但是 this.props.children 由于不是数组, 所以 length 操作会失败

  • React.Children.map(children, function[(thisArg)])
  • React.Children.count(children) 返回数量
  • React.Children.only(children) 判断是否只有一个节点，如果有则返回它，否则此方法会抛出错误
  • React.Children.toArray(children)
  • React.Children.forEach(children, function[(thisArg)])

在 children 里的每个直接子节点上调用一个函数，并将 this 设置为 thisArg。如果 children 是一个数组，它将被遍历并为数组中的每个子节点调用该函数。如果子节点为 null 或是 undefined，则此方法将返回 null 或是 undefined，而不会返回数组

forwardRef

React.forwardRef 会创建一个 React 组件，这个组件能够将其接受的 ref 属性转发到其组件树下的另一个组件中。这种技术并不常见，但在以下两种场景中特别有用

lazy

React.lazy() 允许你定义一个动态加载的组件。这有助于缩减 bundle 的体积，并延迟加载在初次渲染时未用到的组件

默认的 loading 依赖上层的suspense

原理

memo

React.memo 为高阶组件。它与 React.PureComponent 非常相似，但只适用于函数组件，而不适用 class 组件。

Suspense

Suspense通过依赖 componentDidCatch 来控制组件的渲染。实现了组件的异步加载

  • 代码分片
  • 异步获取数据

createPortal

Portal 提供了一种将子节点渲染到存在于父组件以外的 DOM 节点的优秀的方案

startTransition

可以将任务作为低级别任务执行

useDeferredValue

与startTransition作用类似

useTransition

是对比startTransition的hook，可以获取当前是否已经pengding

使用了双缓冲技术，避免了不必要的加载状态
