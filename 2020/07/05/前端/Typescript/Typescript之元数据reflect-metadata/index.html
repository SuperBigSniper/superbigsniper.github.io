Typescript之元数据reflect-metadata

需要注意Reflect.metadata给方法增加元数据是加在原型上的，也就是A.prototype，所以当你找一个实例的元数据也可以找到，因为是根据原型链进行查找的，本质上存在WeakMap<any, Map<any, Map<any, any>>>这样一个结构

先导入 reflect-metadata 这个包，然后在定义的 Post 类的上面用 Reflect.metadata 装饰器添加了一条元数据，role 是这条元数据的名字，admin 是我们给这条数据设置的对应的值。

稍后如果想获取到在类上添加的这些元数据，可以使用 Reflect.getMetadata 方法，role 是元数据的名字，Post 是对应的类的名字。执行上面代码，会返回 admin 这个字符串。

在上面的例子里我们用了 Reflect.metadata 设置的元数据。我们也可以自定义一个装饰器去完成同样的事情。像下面这样：

获取类属性的类型数据

  • 类型元数据使用元数据键”design:type”, 含义为该属性的类型（该属性上有装饰器）
  • 参数类型元数据使用元数据键”design:paramtypes”, 含义为其构造函数的传入参数的类型数组（类上有装饰器）
  • 返回值类型元数据使用元数据键”design:returntype”, 含义为该方法的返回值的类型（方法有装饰器）

只有当一个属性有装饰器，那么Reflect.getMetadata才可以获取它的相关数据

依赖注入

```typescript
type Constructor<T = any> = new (…args: any[]) => T;

const Injectable = (): ClassDecorator => target => {};

class OtherService {
a = 1;
}

@Injectable()
class TestService {
constructor(public readonly otherService: OtherService) {}

testMethod() {
console.log(this.otherService.a);
}
}

const Factory = (target: Constructor): T => {
// 获取所有注入的服务
const providers = Reflect.getMetadata(‘design:paramtypes’, target); // [OtherService]
const args = providers.map((provider: Constructor) => new provider());
return new target(…args);
};

Factory(TestService).testMethod(); // 1
``
