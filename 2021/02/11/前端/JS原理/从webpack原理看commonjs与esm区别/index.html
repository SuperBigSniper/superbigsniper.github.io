commonjs与esm

commonjs作为nodejs的模块管理，esm作为官方标准模块管理，他们除了使用上的区别外还有什么不同呢？

一般的解释有如下几点：

  • CommonJs导出的是变量的一份拷贝，ES6 Module导出的是引用（export default 是特殊的）
  • CommonJs是单个值导出，ES6 Module可以导出多个
  • CommonJs是动态语法可以写在判断里，ES6 Module静态语法只能写在顶层
  • CommonJs的 this 是当前模块，ES6 Module的 this 是 undefined

commonjs介绍

commonjs原理就是闭包，一个文件作为一个模块封装在一个函数中，通过参数传入require,module等参数

一个模块就像下面这样的一个函数

  • exports表示导出模块，是module.exports的简写，可以理解为exports = module.exports
  • require是用来引入模块的方法
  • module表示当前模块的信息
  • __filename模块的文件名
  • __dirname模块的目录名字

require

require除了导入模块外，还有四个静态(方法/属性)

  • require.resolve: 将一个文件的绝对路径解析出来, 例如require.resolve('./1.js')会得到/Users/xxxx/Documents/1.js类似的路径字符串
  • require.main: 表示当前程序运行的入口, 比如1.js调用2.js,2.js又调用3.js, 3.js的入口就是1.js
  • extenstions一个对象，包含加载不同文件的方法，js，json，node
  • cache 存放已经加载的文件，顾名思义：缓存, 还顺便解决了循环引用的问题

module

module是一个包含模块的基本信息的对象

当文件是入口的时候id是一个点.，parent为null，否则为文件的绝对路径,所以我们可以通过这个来判断是否为其它程序调用
