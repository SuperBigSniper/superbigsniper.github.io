flutter基本使用与生命周期

发表于2019-01-25|更新于2022-05-24|flutter
|评论数:

安装

切换版本
Flutter channels/Flutter channels stable

升级项目
flutter upgrade

运行 flutter 项目

flutter run

  • 切换 ios 模式 o
  • 热加载 r
  • 热重启

编译

flutter build ios/macos...

MaterialApp

在主函数 runApp 接受一个 Widget 组件，作为根节点

一般来说接受一个 MaterialApp 组件

MaterialApp 接受下面的参数

  • title:’flutter app’
  • initialRoute:’/‘ 设置初始时显示的页面
  • theme: 主题颜色
  • routes: {“new_page”:(context)=>NewRoute(),} 路由表
  • home：指定首页内容（一般来说是一个组件）

Widget(组件)

widget 是什么

  • Widget 实际上就是 Element 的配置数据，Widget 树实际上是一个配置树，而真正的 UI 渲染树是由 Element 构成；不过，由于 Element 是通过 Widget 生成，所以它们之间有对应关系，所以在表述上，我们可以宽泛的认为 Widget 树就是指 UI 控件树或 UI 渲染树。
  • 一个 Widget 对象可以对应多个 Element 对象

widget 分为以下两种

  • StatelessWidget 无状态组件
  • StatefulWidget 有状态组件

widget，它表示与该 State 实例关联的 widget 实例，由 Flutter framework 动态设置。注意，这种关联并非永久的，因为在应用声明周期中，UI 树上的某一个节点的 widget 实例在重新构建时可能会变化，但 State 实例只会在第一次插入到树中时被创建，当在重新构建时，如果 widget，Flutter framework 会动态设置 State.widget 为新的 widget 实例。

context，它是 BuildContext 类的一个实例，表示构建 widget 的上下文，它是操作 widget 在树中位置的一个句柄，它包含了一些查找、遍历当前 Widget 树的一些方法。每一个 widget 都有一个自己的 context 对象

StatefulWidget
  • 必须实现 createState 方法
  • 对应有个状态管理类，而 createState 的返回值就是这个类的实例
  • 状态管理类通过 widget 获取状态组件的值

和 react 类似，flutter 中也有生命周期

  • initState : 当 Widget 第一次插入到 Widget 树时会被调用(通常在该回调中做一些一次性的操作，如状态初始化、订阅子树的事件通知等)
  • didChangeDependencies : 当 State 对象的依赖发生变化时会被调用
  • didUpdateWidget : 在 widget 重新构建时，Flutter framework 会调用 Widget.canUpdate 来检测 Widget 树中同一位置的新旧节点，然后决定是否需要更新
  • reassemble()：此回调是专门为了开发调试而提供的，在热重载(hot reload)时会被调用，此回调在 Release 模式下永远不会被调用
  • deactivate()：当 State 对象从树中被移除时，会调用此回调
  • dispose()：当 State 对象从树中被永久移除时调用

Scaffold(页面结构)

包含以下组成

  • appBar 头部导航
  • body 页面主题
  • drawer 侧边栏抽屉
  • bottomNavigationBar 底部 tabBar 导航
  • floatingActionButton 右下角浮动按钮区域

appBar

appBar 的值是一个 AppBar 组件

包含下面属性

  • title：Text(‘app 标题’)
  • centerTitle:true 居中标题
  • action:[IconButton(icon:Icon(Icons.search))] 头部右侧动作

drawer

drawer 的值是一个 Drawer 组件
包含下面属性

  • child:ListView()
    • padding
    • children:[]
      • UerAccountsDrawerHeader 一个侧边栏用户信息
      • ListTile 侧边栏选项

bottomNavigationBar

bottomNavigationBar 的值是一个 TabBar 组件

包含下面属性

需要注意的时，底部导航必须包含controller，所以在返回Scaffold之前需要包装，使用DefaultTabController(length:3,child:Scaffold())

  • tabs:[]
    • Tab(icon:Icon(),text())

body

body 的值是一个 TabBarView 组件,表示对应的内容

包含下面的属性

  • children:[]

Container

表示一个容器相当于 div，用于修饰样式

  • decoration: 修饰样式
  • height
  • child：子元素
