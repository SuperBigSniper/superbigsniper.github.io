HTTP

请求类型

  • 请求格式的结构：
    请求头：请求方式、请求的地址和 HTTP 协议版本
    请求行：消息报头，一般用来说明客户端要使用的一些附加信息
    空行： 位于请求行和请求数据之间，空行是必须的。
    请求数据：非必须。
  • HTTP 协议之请求方式
    根据 HTTP 标准，HTTP 请求可以使用多种请求方法。
    HTTP1.0 定义了三种请求方法： GET, POST 和 HEAD 方法。
    HTTP1.1 新增了五种请求方法：OPTIONS, PUT, DELETE, TRACE 和
    CONNECT 方法。
    HEAD 类似 get，只不过返回响应没有具体的内容，用于获取报头
    PUT 从客户端向服务器传送的数据取代指定的文档的内容
    DELETE 请求服务器删除指定的页面
    CONNECT 允许客户端查看服务器的性能
    OPTIONS 允许客户端查看服务器的性能
    TRACE 回显示服务器收到的请求，主要用于测试或判断
  • HTTP 常见响应状态码
    200 OK //客户端请求成功
    400 Bad Request //客户端请求有语法错误，不能被服务器所理
    解
    401 Unauthorized //请求未经授权，这个状态代码必须和 WWW-Authenticate 报头域一起使用
    403 Forbidden //服务器收到请求，但是拒绝提供服务
    404 Not Found //请求资源不存在，eg：输入了错误的 URL
    500 Internal Server Error //服务器发生不可预期的错误
    503 Server Unavailable //服务器当前不能处理客户端的请求，一段时
    间后可能恢复正常

生命周期
service
request 方法
  1. //获取请求方式
    String method = req.getMethod();
  2. //获取请求 URL
    StringBuffer url=req.getRequestURL();
  3. //获取协议
    String h=req.getScheme();
  4. //获取指定的请求行信息
    String value=req.getHeader(“aaa”);
  5. //设置请求的编码格式：
    req.setCharacterEncoding(“utf-8”)
  6. //获取所有的请求行的键的枚举
response 方法

使用：

  • 设置响应头
    setHeader(String name,String value);//在响应头中添加响应信息，但是同键会覆盖
  • addHeader(String name,String value);//在响应头中添加响应信息，但是不会覆盖。
  •       设置响应状态
    sendError(int num,String msg);//自定义响应状态码。
  •       设置响应实体
    resp.getWrite().write(String str);//响应具体的数据给浏览器
  •   设置响应编码格式：
    resp.setContentType(“text/html;charset=utf-8”);
  •   总结：
    service 请求处理代码流程：
    设置响应编码格式
    获取请求数据
    处理请求数据
    数据库操作（MVC 思想）
    响应处理结果
请求转发

  • req.getRequestDispatcher(“要转发的地址”).forward(req,resp)
    地址：相对路径，直接书写 servlet 的别名

  • req.setAttribute(“str”,”用户名错误”)
    在 req 中设置属性 str，之后请求转发都会带有 str

  • req.getAttribute(“str”)
    获取属性值，返回的是一个对象

  • 重定向

    • resp.sendRedirect(“路径”) 重定向到网站

  • cookie

session
  • 获取和设置 session
    HttpSession hs = req.getSession();
    如果没有 session 会自动设置 session,并返回这个 session
    销毁时间默认 30 分钟，如果重新发出请求回重置为 30 分钟
    浏览器关闭即失效
  • 获取 session 的 ID
    hs.getId()
  • 设置 session 存储时间
    hs.setMaxInactiveInterval(5) 默认以秒为单位
    session 对象失效了，会自动重新创建一个 session 对象
  • 设置 session 强制失效
    hs.invalidate()
  • 存储数据
    hs.setAttribute(“name”,value);
  • 获取数据
    hs.getAttribute(“name”)

servletContext (全局环境，负责各个 servlet 之间的数据传递)

  • 获取 ServletContext 对象
    1. ServletContext sc = this.getServletContext()
    2. ServletContext sc2 = this.getServletConfig().getServletContext()
    3. ServletContext sc3 = req.getSession().getServletContext();
      上面三种方法都一样
  • 设置数据存储
    sc.setAttribute(“name”,value)
  • 获取数据
    sc.getAttribute(“name”)
  • 获取 web.xml 中的全局配置数据
    String str = sc.getInitParameter(“name”)
    sc.getInitParameterNames() 返回键名的枚举
  • 获取根目录下的资源的绝对路径
    String path = sc.getRealPath(“/doc/1.txt”) webroot 下的资源的绝对路径
  • 获取项目根目录下资源的流对象
    InputStream is = sc.getResourceAsStream(“/doc/1.txt”)

ServletConfig(每个 servlet 都有一个这样的对象，用于获取 web.xml 中的配置)

  • 获取 ServletConfig 对象
    ServletConfig sc = this.getServletConfig();
  • 获取 web.xml 中的配置数据
    sc.getInitParameter(“config)

过滤器

作用与概念
  • 作用
    1. 与 node 的中间件差不多，就是进出的数据需要进行过滤
    2. 对服务器资源进行管理保护
    3. 保护 servlet
使用

  • 创建一个实现了 Filter 接口的普通 java 类

    • init 方法 初始化，服务器启动即执行，资源初始化
    • doFilter 方法
      拦截请求的方法，可以对资源进行管理
      需要手动放行，chain.doFilter(request, response)
    • destory 方法 服务器关闭的时候执行

  • 在 web.xml 中配置过滤器

    • 统一编码格式
    • session 管理
