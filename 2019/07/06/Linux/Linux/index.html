用户相关

useradd 选项 用户名

useradd –d /home/sam zzz 指定用户zzz的家目录为/home/sam

-c comment 指定一段注释性描述。
-d 目录 指定用户主目录，如果此目录不存在，则同时使用-m选项，可以创建主目录。
-g 用户组 指定用户所属的用户组。
-G 用户组，用户组 指定用户所属的附加组。
-s Shell文件 指定用户的登录Shell。
-u 用户号 指定用户的用户号，如果同时有-o选项，则可以重复使用其他用户的标识号。

userdel 选项 用户名

userdel -r sam 添加了-r选项会把用户的家目录一起删除

usermod 选项 用户名 （修改用户相关的属性，与添加用户的参数相同）

例如：添加一个额外的组给用户，可以使用如下命令：sudo usermod -G root sniper

使用id sniper就会得到用户id=1000(sniper) 组id=1000(sniper) 组=1000(sniper),0(root)

passwd 选项 用户名

-l 锁定口令，即禁用账号。
-u 口令解锁。
-d 使账号无口令。
-f 强迫用户下次登录时修改口令。

groupadd 选项 用户组

-g GID 指定新用户组的组标识号（GID）。
-o 一般与-g选项同时使用，表示新用户组的GID可以与系统已有用户组的GID相同。

例如 groupadd group1

groupdel 用户组

groupmod 选项 用户组

/etc/passwd文件是用户管理工作涉及的最重要的一个文件(/etc/shadow, /etc/group也很重要)

gpasswd 用于管理用户与用户组的关系

-a：添加用户到组；
-d：从组删除用户；
-A：指定管理员；
-M：指定组成员和-A的用途差不多；
-r：删除密码；
-R：限制用户登入组，只有组中的成员才可以用newgrp加入该组。

用户名:口令:用户标识号:组标识号:注释性描述:主目录:登录Shell

改变文件属性与权限
  • chgrp ：改变文件所属群组
  • chown ：改变文件拥有者与群组 chown -R root:root /home/sam // -R表示递归，root:root前面是用户，后面是群组
  • chmod ：改变文件的权限, SUID, SGID, SBIT 等等的特性

都有-R属性

文件内容查阅
  • cat 由第一行开始显示文件内容
  • tac 从最后一行开始显示，可以看出 tac 是 cat 的倒着写！
  • nl 显示的时候，顺道输出行号！
  • more 一页一页的显示文件内容
  • less 与 more 类似，但是比 more 更好的是，他可以往前翻页！
  • head 只看头几行
  • tail 只看尾巴几行
  • od 以二进制的方式读取文件内容！
解压缩
  • c 表示压缩，打包(不会压缩)
  • f 表示后面接文件名(表示输出到文件，而不是打印到控制台)
  • v 参数表示显示执行过程
  • z 打包为gz格式
  • j 打包为bz2格式（压缩比率更高，但是更慢）
  • x 解压缩

必须把f放在最后面，因为f后面要接参数

tar -zcvf xxx.tar.gz 目录

tar -jcvf xxx.tar.bz2 目录 bz2 压缩

将目录添加到 xxx.tar.gz

解压

tar -zxvf xxx.tar.gz 目录 解压缩

tar -jxvf xxx.tar.bz2 目录 解压缩 bz2

显示进程

ps

  • e 显示所有shell启动的进程
  • f 显示uid, pid, parent pid, cpu使用情况等等
  • L 显示线程

ps -eLf

systemctl

systemctl [command] [unit]

配置文件一般在 /usr/lib/systemd/system

  • command 主要有：
  • start：立刻启动后面接的 unit。
  • stop：立刻关闭后面接的 unit。
  • restart：立刻关闭后启动后面接的 unit，亦即执行 stop 再 start 的意思。
  • reload：不关闭 unit 的情况下，重新载入配置文件，让设置生效。
  • enable：设置下次开机时，后面接的 unit 会被启动。
  • disable：设置下次开机时，后面接的 unit 不会被启动。
  • status：目前后面接的这个 unit 的状态，会列出有没有正在执行、开机时是否启动等信息。
  • is-active：目前有没有正在运行中。
  • is-enabled：开机时有没有默认要启用这个 unit。
  • kill ：不要被 kill 这个名字吓着了，它其实是向运行 unit 的进程发送信号。
  • show：列出 unit 的配置。
  • mask：注销 unit，注销后你就无法启动这个 unit 了。
  • unmask：取消对 unit 的注销。

systemctl get-default 获取默认系统启动的级别
systemctl set-default mutil-user.target 设置默认系统启动的级别
