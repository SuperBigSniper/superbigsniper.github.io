lock

java.util.concurrent.locks.Lock 机制提供了比synchronized代码块和synchronized方法更广泛的锁定操作,
同步代码块/同步方法具有的功能 Lock 都有,除此之外更强大,更体现面向对象

Lock 锁也称同步锁，加锁与释放锁方法化了，如下：

  • public void lock() :加同步锁。
  • public void unlock() :释放同步锁。

实战

在写的过程中其实很容易遇到，没有锁住对象，只有一个线程在跑的尴尬

  1. 竞态资源一定要放入synchronized方法或者块中，否则就会出现问题
  2. 当你要操作竞态资源时，一定要先判断能否能否进入，如果在你操作竞争之后才进行判断也会导致你的锁失效(这里其实不是失效，是逻辑错误)
  3. 还需要注意synchronized不能在 while 循环外，这也会导致只有一个线程跑完，其它线程才跑，因为 while 循序没有结束，就不会自动释放锁，就会导致一个线程一直跑

同步控制之 wait join notify

这三个方法都是 java 原生提供用于线程同步控制的，都需要在synchronized块中进行调用

  1. wait 表示将当前线程停止，将控制权交给其它线程
  2. notify 表示唤醒其它线程，还有 motifyAll 唤醒其它所有线程
  3. join 表示等待调用 join 方法的线程执行完成

下面是通过wait与notify简单实现一个生产者消费者模型，当然这个地方不是特别高效，因为不管是生产者还是消费者都会获取同一个锁对象，一般来说还会有一个缓冲区

join的应用场景: 当多个线程并发执行，但是需要线程的执行顺序，比如：一个爬虫，并发爬取小说章节，这个小说有很多几千章节，爬取下来后，我肯定需要按顺序去读，这个时候join的效果就出来了

还有类似的Thread.yield()方法：暂停当前正在执行的线程对象，并执行其他线程，有可能执行 yield 方法执行后还是自己执行
