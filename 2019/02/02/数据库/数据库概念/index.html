数据库概念

发表于2019-02-02|更新于2022-04-06|数据库
|评论数:

三大范式

  • 第一范式：当关系模式R的所有属性都不能在分解为更基本的数据单位时，称R是满足第一范式的，简记为1NF。（其实就是指列必须在这个项目中满足不可拆分，即列不可拆分）

  • 第二范式：如果关系模式R满足第一范式，并且R得所有非主属性都完全依赖于R的每一个候选关键属性，称R满足第二范式，简记为2NF。（就是指一个数据可以根据一个唯一标识来确定，即唯一标识）

  • 第三范式：设R是一个满足第一范式条件的关系模式，X是R的任意属性集，如果X非传递依赖于R的任意一个候选关键字，称R满足第三范式，简记为3NF.（引用其它表关系时，只能引用主键）

约束

  • 主键 primary key
  • 非空 not null
  • 唯一 unique
  • 默认值 default
  • 外键 foreign key

连接join

  • inner join 内连接：表A inner join 表B：表A与表B匹配的行会出现在结果中
  • left join 左连接：表A left join 表B：表A与表B匹配的行会出现在结果中，外加表A中独有的数据，未对应的数据使用null填充
  • right join 右连接：表A right join 表B：表A与表B匹配的行会出现在结果中，外加表B中独有的数据，未对应的数据使用null填充
  • natural join自然连接：(相同字段会合并)自然连接是一种特殊的等值连接，他要求两个关系表中进行比较的必须是相同的属性列，无须添加连接条件，并且在结果中消除重复的属性列
